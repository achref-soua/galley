# Localization

Galley's UI strings are externalised so the interface can be translated without
touching components. English ships today; the scaffolding is ready for more
locales.

## How it works

- **Catalogs** are flat maps of dot-namespaced keys to template strings, one per
  locale (`src/lib/locales/en.ts`).
- **`i18n.ts`** provides `interpolate` (`{named}` placeholders), `translate`
  (catalog → English fallback → the key itself), and the `I18n` translator class.
- **`i18n-store.ts`** exposes the shared instance and the `t(key, params?)`
  helper components call.

```ts
import { t } from './i18n-store';

t('onboarding.import.title'); // "Import your Overleaf project"
t('app.name'); // "Galley"
```

## Adding a locale

1. Create `src/lib/locales/<tag>.ts` mirroring the keys in `en.ts`.
2. Register it in `i18n-store.ts`'s `catalogs` map and add the tag to `Locale`.
3. Any missing key automatically falls back to English.

## Adopting `t()` in a component

Replace a literal string with `t('namespace.key')`. The onboarding tour is the
first fully-localised surface; other surfaces are migrated incrementally. Keys are
grouped by surface (`app.*`, `onboarding.*`, `a11y.*`, …).
