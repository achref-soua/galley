# ADR-0035 — Bug sweep: preview, diagnostics, dock, and an honest proof (v0.9.4)

**Status:** Accepted  
**Date:** 2026-06-21  
**Deciders:** Achref Soua

---

## Context

Installing v0.9.3 and using it as a real user surfaced a cluster of defects that
the unit and e2e suites missed because they exercise the UI under jsdom and stub
backends — none of which run the real PDF.js worker, a real TeX engine, or a real
pointer drag. The four that mattered: the PDF preview broke on its second render,
zoom did nothing, compile warnings were not actionable, and the editor's bottom
panel could not be resized. The screenshots also hid the preview because the web
build's stub proof was a blank page.

This is a dedicated find-and-fix release before v1.0.0.

## Decisions

### 1. Copy the PDF bytes per render rather than holding them immutably

`pdfjs.getDocument({ data })` transfers `data`'s `ArrayBuffer` to the worker,
detaching it. The preview holds the compile result immutably and re-renders it on
every page turn and zoom change, so the second render hit a detached buffer and
threw. The renderer (`pdf.ts`) now passes `data.slice()` — a fresh copy — to PDF.js
each call, so the worker detaches the copy and the caller's bytes stay intact. The
fix lives in the thin PDF adapter, not the component; the regression test drives a
mock that detaches what it receives and asserts two renders from the same bytes
succeed. This also fixes zoom, which was the same bug triggered by a scale change.

### 2. Package-note classes as first-class diagnostics, with expandable detail

The log parser (`galley_core::diagnostics`) gained three categories — the
`inputenc`-ignored note, an empty `thebibliography`, and "rerun to get
cross-references right" — each with a fixed severity, a plain-language explanation,
and a fix tip, so a warning a user actually hits reads as advice rather than raw
TeX. The problems panel now renders each diagnostic as a disclosure: the
explanation up front, and an expandable detail with the raw log line and a
jump-to-source button. Every entry expands, even one with no line number, so an
unlocated warning is still inspectable. The classification stays in pure Rust (the
only place it earns real coverage); the panel only renders.

### 3. Give the bottom dock a persisted, draggable height

The layout model (`@galley/ui-kit`) gained a `bottomPanelHeight` clamped to
`[96, 640]`, and the dock below the editor (problems, outline, review) is wrapped
in a fixed-height, scrollable container with a horizontal resize handle above it.
The `Resizer` component grew an `orientation` prop so the same focusable handle
drives both axes (drag, plus arrow-key nudges). `bottomPanelHeight` is optional in
the persisted shape, so a layout saved before this release keeps its pane widths
and simply falls back to the default dock height instead of being discarded.

### 4. Embed a real typeset proof as the web build's stub compile result

The engine-less browser/dev build returned a blank one-page PDF, which made the
live preview look empty and forced the screenshots to hide it. We now embed a
genuine one-page proof — typeset from a representative sample document by the
embedded Tectonic engine (regenerated by the `writes_the_sample_proof` helper in
`crates/galley-compile/tests/real_compile.rs`) — as the in-memory compile result.
The live preview, the Playwright e2e, and the regenerated screenshots/demo now show
an honest editor-and-proof view. The packaged app is unchanged: it compiles every
document for real.

## Consequences

- Paging and zoom work in the packaged app; the second render no longer throws.
- Warnings are actionable and every diagnostic is inspectable, located or not.
- The dock can be shrunk to reclaim editor space, and the choice persists.
- Screenshots and the demo show the real proof beside the source, and stay honest
  because they are generated from the same sample the embedded proof was typeset
  from. The embedded proof adds ~15 KB to the web bundle, well within the budget.
- 100% line/branch/function coverage holds across the changed Rust and TypeScript;
  e2e gained regressions for the zoom re-render, the expandable jump, and the dock
  drag.
