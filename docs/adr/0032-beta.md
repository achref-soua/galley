# ADR-0032 — Beta: crash reporting, feedback, privacy (v0.8.0)

**Status:** Accepted  
**Date:** 2026-06-21  
**Deciders:** Achref Soua

---

## Context

v0.8.0 is the beta milestone (§7): feature freeze, installers in cohort hands on
all three OSes, opt-in privacy-respecting crash reporting, a feedback channel, and
a bug burndown. Galley's privacy stance (§8.4) is strict — off by default, opt-in,
anonymised — so the beta must put the consent model in place before any reporting
exists.

## Decisions

### 1. Feature freeze for the beta line

No new product surface lands in v0.8.0 beyond the beta scaffolding (crash
reporting consent, feedback). Subsequent work is bug-fixing and stabilisation
toward the v0.9.0 release candidate.

### 2. Crash reporting is consent-first, anonymised, and send-deferred

The consent toggle (`PrivacyPrefsStore`, off by default) and the anonymising
payload builder (`crash-report.ts`) ship now. `buildCrashReport` returns `null`
without consent, so nothing can be produced by accident; with consent it strips
absolute paths/URLs from the stack and never includes document content. The actual
network send has **no endpoint** in this release — deferred to post-beta per §8.4 —
the same wired-but-dormant posture as code signing and the updater. This puts the
privacy model first.

### 3. Feedback is a pre-filled issue the user submits

`feedback.ts` builds a GitHub "new issue" URL stamped with the version and OS.
Nothing is sent automatically; **Settings → About → Send feedback…** opens it for
the user to review and post.

### 4. The version is single-sourced from package.json

The About panel and the feedback/crash context read the version from
`package.json` (imported in the app), so the displayed version can never drift
from the released one.

## Consequences

- Beta users can opt into crash reporting and send feedback, with a privacy notice
  (`docs/privacy.md`) shipping alongside.
- The consent + anonymisation model is in place before any telemetry, so enabling
  the send later is a configuration step, not a privacy redesign.
- Cross-OS installers are built per OS (Linux here; Windows/macOS on their
  platforms) and handed to the cohort with checksums.
